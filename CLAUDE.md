To run Convex backend locally alongside the dev server, run `npx convex dev` in a separate terminal.

## Architecture

**Stack**: React 19 + TanStack Router + Convex (serverless backend) + Tailwind CSS v4 + Vite

**Data flow**: All backend logic lives in `convex/`. Components call Convex via `useQuery` and `useMutation` hooks (from `convex/react`). There is no REST API — Convex handles real-time sync automatically.

**Routing**: TanStack Router with file-based routing in `src/routes/`. `src/routeTree.gen.ts` is auto-generated — do not edit. The root layout (`src/routes/__root.tsx`) wraps all routes with `AppShell`, which renders `Sidebar` (desktop) and `BottomTabBar` (mobile).

**Path aliases**: `#/*` → `src/*`, `@convex/*` → `convex/*`

## Convex Backend (`convex/`)

- `schema.ts` — defines 5 tables: `exercises`, `workoutSessions`, `sets`, `oneRepMaxes`, `routines`
- Each module (`exercises.ts`, `workoutSessions.ts`, `sets.ts`, etc.) exports named `query` and `mutation` functions
- `convex/_generated/` is auto-generated by the Convex CLI — never edit these files
- `convex/lib/oneRepMax.ts` — Epley formula shared with frontend at `src/lib/oneRepMax.ts`

Key schema relationships:
- `sets` → `workoutSessions` (sessionId) → `exercises` (exerciseId)
- `oneRepMaxes` → `exercises` (exerciseId); recalculated on session delete
- `routines` contain an embedded array of exercise references (not a join table)

## Frontend (`src/`)

- `src/components/ui/` — base UI primitives (shadcn/ui + Base UI)
- `src/components/session/` — active workout session components
- `src/lib/utils.ts` — `cn()` helper for conditional Tailwind classes

## Code Style

Biome enforces: 100-char line width, 2-space indent, single quotes, strict lint rules. Run `npm run lint` before marking work done.

TypeScript strict mode is enabled. Convex function argument types come from `convex/_generated/api` — use the generated types, not manual ones.
